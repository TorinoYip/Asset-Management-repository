# Asset Intellgent 网页

这是 Asset Intellgent 的静态网页，用于验证资产管理看板的五页信息架构、页面层级与跨页导航。

## 打开方式

直接打开 `index.html` 即可；也可以从仓库根目录启动任意静态文件服务：

```bash
python3 -m http.server 8000
```

然后访问 `http://localhost:8000/apps/web/`。

## 已实现

- `#overview`：资产看板，采用“2 个一级主区 + 3 个二级概况”的结构；
- `#revenue`：营收总览；
- `#types`：按类型分析；
- `#stations`：按站点分析；
- `#mengxi`：蒙西日报入口，本轮仅保留页面骨架；
- 左侧导航与返回入口；
- 经营摘要向营收总览下钻；
- 截止月份、省份、电站和资产类型筛选上下文；
- 四项全局筛选会同步更新资产看板与营收总览中的卡片、地图、趋势和表格；
- 仓库内置的中国省级轮廓地图，不依赖外部地图或样式服务；
- 河北、内蒙古、山东、浙江四个省份各一个资产点；悬停或聚焦时从点位向上延伸引导线，在地图内显示半透明省内资产板；
- 省内资产板采用高密度四列信息，仅展示资产名称、YTD 毛利、目标达成率与状态；
- 资产构成表与可筛选、可多选的资产明细横表；
- 月份下拉对比表；
- YTD / 具体月份的同类资产横向比较；
- 风电单机与储能单站的同级运营指标比较；
- 站点经营、运营、运维摘要与结算单拆解。

## 文件边界

```text
apps/web/
├── index.html                 # 应用外壳、导航与全局筛选
├── assets/china-map.svg       # 本地地图图形资产
├── src/
│   ├── core/                  # 启动、路由与事件协调
│   ├── data/                  # 组合、站点和指标数据
│   ├── domain/                # 筛选、聚合和财务口径计算
│   ├── pages/                 # 五页模板及页面渲染器
│   └── shared/                # 通用 DOM 与格式化工具
└── styles/
    ├── core.css               # 基础视觉、应用外壳和公共组件
    ├── dashboard.css          # 看板页面和分析页面
    └── map.css                # 中国地图及省份资产表
```

`index.html` 不再承载五页正文，`app.js` 仅作为兼容标记。页面使用浏览器原生脚本按依赖顺序加载，因此既支持直接打开，也不改变 GitHub Pages 的静态发布方式。

指标名称与层级参考信息架构及指标字典。正式开发时应以接口替换 `src/data/portfolio.js` 中的数据，并继续复用 `src/domain/portfolio.js` 中的统一口径。

## 地图使用说明

当前 SVG 仅用于内部原型的交互和布局验证。正式公开发布地图时，应从自然资源主管部门的标准地图服务获取合规底图，不修改重要地理要素，并按要求标注审图号。
