# V2 信息架构

## 设计目标

V1 已覆盖首页总览、经营看板、运营看板，以及风机、储能、运维等指标。本次 V2 不改变业务范围，重点解决三个问题：

1. 将“页面导航”“指标口径”“数据粒度”分开表达。
2. 建立从集团总览到站点、单机或单储的逐级钻取路径。
3. 统一风机与储能的页面骨架，使后续组件和数据接口可复用。

## 一级导航

| 页面 | 决策问题 | 默认粒度 | 主要去向 |
|---|---|---|---|
| 首页总览 | 当前资产规模、状态和总体经营表现如何？ | 集团 / 当前筛选范围 | 经营看板、运营看板、站点详情 |
| 经营看板 | 收入、成本、毛利是否达到目标，差距来自哪里？ | YTD + 月度 | 站点经营详情 |
| 运营看板 | 哪些设备或站点影响发电、可用率与损失？ | 月度 / 站点 | 风机详情、储能详情、运维详情 |

## 页面层级

```text
资产看板
├── 首页总览
│   ├── 资产概览
│   ├── 经营摘要
│   └── 运营摘要
├── 经营看板
│   ├── 集团经营总览
│   ├── 站点经营排名
│   └── 站点经营详情
└── 运营看板
    ├── 风机运营
    │   ├── 站点总览
    │   ├── 单机月度
    │   └── 运维指标
    └── 储能运营
        ├── 站点总览
        ├── 单储月度
        └── 运维指标
```

## 关键导览关系

- 首页资产卡片 → 对应资产类型的运营看板，并继承区域、公司、站点和时间筛选。
- 首页收入、成本、毛利卡片 → 经营看板，并自动定位相应指标。
- 经营排名或站点对比 → 站点经营详情。
- 风机 / 储能站点排名 → 对应站点总览。
- 站点总览 → 单机 / 单储月度明细。
- 异常指标、损失率、MTTR → 运维详情，并携带设备与时间上下文。
- 面包屑或返回操作必须保留筛选上下文，避免用户重新选择。

## 统一页面骨架

1. 顶部：页面标题、数据更新时间、全局筛选器。
2. 第一屏：核心 KPI 卡片，默认展示实际值、目标值、差距和同比/环比。
3. 第二屏：趋势与构成，用于回答“何时发生、由什么组成”。
4. 第三屏：排名或明细表，用于定位“哪个站点或设备导致”。
5. 钻取页：设备或站点详情、异常说明、关联工单。

## V2 边界

- 当前 draw.io 文件表达页面关系与导览，不代替最终 UI 高保真稿。
- 指标公式、单位、粒度和刷新频率以指标字典为准。
- V1 中的 C1–C4 等临时口径先保留为“待确认”，不直接固化进代码。
