# V2 信息架构

## 1. 产品定位

本产品是面向管理层与资产经营人员的内部资产管理透视看板，不是 SCADA 实时监控系统，也不是完整 EAM/工单系统。V2 目标是清晰呈现资产规模、经营结果和运营表现。

## 2. 数据与时间语义

- 最小时间粒度：月度。
- 页面统一显示“截至 YYYY 年 MM 月”。
- 实际 YTD：年初至所选月份的实际累计值。
- 累计目标：年初至所选月份的计划累计值。
- 全年目标：完整年度目标值。
- 日度日报属于独立系统，不进入本看板路由与页面层级。

## 3. 资产对象与同级原则

资产类型包括风电和储能。进入资产月度分析层后：

| 资产类型 | 单体分析对象 | 时间粒度 |
|---|---|---|
| 风电 | 单台风机 | 月度 |
| 储能 | 单个储能电站（单站） | 月度 |

风电单机与储能单站是同级资产对象，必须具备相同等级的功能：

- 资产概览；
- 月度指标；
- 实际值、目标值和差距；
- 状态展示；
- 排名与横向对比；
- 筛选与选择；
- 详情入口。

二者只在专业指标上不同，不在页面层级和交互能力上形成高低差异。风电场/项目和区域仅作为风机的聚合维度；储能单站也可以按区域或项目组合聚合。

## 4. 一级页面

### 4.1 资产总览

回答“拥有哪些资产、规模多大、当前总体状态如何”。

- 组合 KPI：资产数量、风电装机容量、储能功率/容量、YTD 发电量、YTD 充放电量、资产状态。
- 资产地图：按经纬度展示风电项目和储能单站；颜色表达资产类型/状态，标记大小表达容量。
- 组合构成：区域、资产类型、状态分布。
- 经营摘要：收入、成本、毛利的实际 YTD、累计目标与差距。
- 运营摘要：发电/充放电、可利用率、效率、限电与损失。

### 4.2 经营看板

风电与储能使用同一页面骨架，通过 Tab 切换：

1. YTD KPI：实际、累计目标、差距、累计目标完成率、全年目标、全年目标完成进度。
2. 月度趋势：月度实际；累计实际与累计目标。
3. 经营构成：收入、成本、毛利及已有分项。
4. 资产排名：风电按现有经营核算对象展示，储能按单站展示；字段、排序、状态和交互能力一致。
5. 后续能力：同比、环比和详情查看。

### 4.3 运营看板

风电与储能同样使用统一骨架：

- 顶部：资产类型 Tab 和对应运营 KPI。
- 中部：聚合表现和资产排名。
- 底部：同级资产月度对比。
  - 风电：单机 × 月份；
  - 储能：单站 × 月份。
- 运维异常摘要：MTTR、计划/非计划损失和异常状态等，只作为运营页中的次级模块。

## 5. 交互与信息流

### 5.1 全局上下文

后续全局筛选包括：年份、截至月份、区域/项目公司、资产类型和资产对象。页面跳转必须继承筛选上下文。

### 5.2 当前信息流

```text
资产地图/摘要
├── 经营摘要 → 经营看板
└── 运营摘要 → 运营看板

经营看板
├── 风电 Tab → 风电 YTD、趋势、排名与对比
└── 储能 Tab → 储能 YTD、趋势、排名与对比

运营看板
├── 风电 Tab → 单机月度排名与对比
├── 储能 Tab → 单站月度排名与对比
└── 运维摘要 → 当前仅呈现异常概况
```

## 6. 当前边界

V2 当前只负责组合、经营和运营信息的清晰呈现，不建设日度日报、根因诊断、工单执行、责任认定和财务损失闭环。
